# UI Design Brief

Use this when handing the product plan to a UI design workflow.

## Brief Format

Provide:

- Product name or working title.
- Platform and component system.
- Product positioning.
- Target user.
- Core loop.
- First-level pages.
- Second-level pages.
- Screen-by-screen content requirements.
- Key data examples.
- Visual tone.
- Important states.
- Constraints and non-goals.

## Screen Content Template

For each screen:

```text
Screen name:
Level: first-level or second-level
Parent or entry point:
Purpose:
Primary content:
Primary action:
Secondary actions:
Data examples:
State notes:
```

## Handoff Rule

Only hand off to UI design after the page list includes enough second-level screens to complete the product loop.

For `$sanbao-ui-design`, request a pure design board when appropriate:

- No external flow arrows.
- No explanatory callouts outside screens.
- No product notes on the canvas.
- Multiple app screens only, using real app UI titles and content.
